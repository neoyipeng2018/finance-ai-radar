Plan: Closed-loop finance AI aggregator

Workflow, Review Queue, Public Dataset Expansion, and Self-Improving Agent Loop

Goal: turn Finance AI Radar from a curated static homepage into a learning system: ingest public finance/NLP sources, stage candidates in a review queue, publish only rights-safe reviewed items, measure what users click and read, run a daily reflection agent over metrics, update AGENTS.md with lessons, and have an implementation agent execute appropriately sized, tested improvements, including larger refactors when the evidence and verification gates justify them.

North star: every day the website learns which AI-in-finance themes are useful to readers and converts that learning into better source acquisition, ranking, UX, monetization, and editorial guidance.

1. System Architecture

Source Ingestion
Review Queue
Publication

Current baseline

  • Next.js app at finance-ai-radar/.
  • Curated seed items live in data/sourceItems.ts.
  • Human-auditable source ledger lives in data/source_ledger.tsv.
  • Library tests validate ranking, search, source counts, theme briefs, and editorial metrics.

Target design

  • Queue-first ingestion: crawlers write drafts, not homepage items.
  • Dataset and Hugging Face expansion: Kaggle, Hugging Face datasets, Hugging Face models, Hugging Face papers, data.gov, SEC, FRED, World Bank, BIS, CFPB, FINRA, OCC, FDIC, and central-bank datasets become first-class content.
  • First-party analytics: click, dwell, scroll depth, filter, theme, sponsor, and newsletter events are logged to a small events table.
  • Daily reflection: a Hermes cron job reads metrics and writes evidence-backed edits to AGENTS.md.
  • Implementation agent: a second job or chained agent executes the right-sized improvement with tests and build verification; small changes are preferred, but bigger changes are allowed when metrics reveal a structural bottleneck.
Important safety gate: the implementation agent must not directly deploy arbitrary changes from metrics. It should create a branch or local patch, run tests, record the decision in data/daily_reflections.tsv, and only ship changes that pass explicit quality gates. Larger changes are allowed when the reflection clearly states the evidence, scope, rollback plan, and verification commands.

3. Review Queue Workflow

The queue should make curation scalable without polluting the homepage. Everything discovered by crawlers lands as a candidate. Publishing requires the editorial rule: source link + why it matters + caveat + license note.

  • ✅ Is there a caveat: leakage, bias, marketing, reproducibility, data rights, or compliance?

4.2 Kaggle ingestion script

Use Kaggle API when credentials exist. If credentials are absent, the script should skip gracefully and log setup instructions.

# scripts/ingest_kaggle.py
import csv
import json
import subprocess
from datetime import datetime, timezone
from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
QUEUE = ROOT / "data" / "review_queue.tsv"

QUERIES = [
    "financial news nlp",
    "finance sentiment",
    "sec filings",
    "credit default",
    "loan default",
    "credit card fraud",
    "earnings call transcript",
]

FIELDS = [
    "candidate_id", "source_type", "title", "url", "publisher", "discovered_at",
    "relevance_reason", "license_guess", "status", "reviewer_notes"
]

def run_kaggle(query: str) -> list[dict[str, str]]:
    result = subprocess.run(
        ["kaggle", "datasets", "list", "--search", query, "--csv"],
        text=True,
        capture_output=True,
        check=False,
    )
    if result.returncode != 0:
        print(json.dumps({"query": query, "status": "skipped", "stderr": result.stderr[:300]}))
        return []
    rows = csv.DictReader(result.stdout.splitlines())
    return list(rows)[:8]

def append_candidate(row: dict[str, str], query: str) -> dict[str, str]:
    ref = row.get("ref", "").strip()
    title = row.get("title", ref).strip()
    return {
        "candidate_id": f"kaggle-{ref.replace('/', '-').lower()}",
        "source_type": "kaggle",
        "title": title,
        "url": f"https://www.kaggle.com/datasets/{ref}",
        "publisher": ref.split("/")[0] if "/" in ref else "Kaggle",
        "discovered_at": datetime.now(timezone.utc).isoformat(),
        "relevance_reason": f"Matched Kaggle query '{query}' for finance/NLP dataset monitoring.",
        "license_guess": row.get("licenseName", "Check Kaggle dataset page before reuse") or "Check Kaggle dataset page before reuse",
        "status": "candidate",
        "reviewer_notes": "Needs provenance, license, NLP use case, leakage risk, and quality review.",
    }

def main() -> None:
    QUEUE.parent.mkdir(parents=True, exist_ok=True)
    existing = set()
    if QUEUE.exists():
        with QUEUE.open(newline="") as f:
            existing = {r["candidate_id"] for r in csv.DictReader(f, delimiter="\t")}
    write_header = not QUEUE.exists()
    with QUEUE.open("a", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=FIELDS, delimiter="\t")
        if write_header:
            writer.writeheader()
        for query in QUERIES:
            for row in run_kaggle(query):
                candidate = append_candidate(row, query)
                if candidate["candidate_id"] not in existing:
                    writer.writerow(candidate)
                    existing.add(candidate["candidate_id"])

if __name__ == "__main__":
    main()

4.3 Hugging Face dataset ingestion script

# scripts/ingest_hf_datasets.py
import csv
from datetime import datetime, timezone
from pathlib import Path
from urllib.parse import quote
from urllib.request import Request, urlopen
import json

ROOT = Path(__file__).resolve().parents[1]
QUEUE = ROOT / "data" / "review_queue.tsv"
FIELDS = ["candidate_id", "source_type", "title", "url", "publisher", "discovered_at", "relevance_reason", "license_guess", "status", "reviewer_notes"]
QUERIES = ["finance nlp", "financial sentiment", "sec filings", "financial question answering", "earnings call"]

def search(query: str) -> list[dict]:
    url = f"https://huggingface.co/api/datasets?search={quote(query)}&limit=10&full=true"
    req = Request(url, headers={"User-Agent": "finance-ai-radar/0.1"})
    with urlopen(req, timeout=30) as response:
        return json.loads(response.read().decode("utf-8"))

def candidate(dataset: dict, query: str) -> dict[str, str]:
    dataset_id = dataset.get("id", "")
    card = dataset.get("cardData") or {}
    license_value = card.get("license") or "Check dataset card and original source"
    return {
        "candidate_id": f"hf-{dataset_id.replace('/', '-').lower()}",
        "source_type": "huggingface_dataset",
        "title": dataset_id,
        "url": f"https://huggingface.co/datasets/{dataset_id}",
        "publisher": dataset_id.split("/")[0] if "/" in dataset_id else "Hugging Face",
        "discovered_at": datetime.now(timezone.utc).isoformat(),
        "relevance_reason": f"Matched Hugging Face dataset query '{query}' for finance/NLP monitoring.",
        "license_guess": str(license_value),
        "status": "candidate",
        "reviewer_notes": "Needs dataset-card review, source provenance, task fit, and leakage-risk note.",
    }

def main() -> None:
    QUEUE.parent.mkdir(parents=True, exist_ok=True)
    existing = set()
    if QUEUE.exists():
        with QUEUE.open(newline="") as f:
            existing = {r["candidate_id"] for r in csv.DictReader(f, delimiter="\t")}
    with QUEUE.open("a", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=FIELDS, delimiter="\t")
        if not existing:
            writer.writeheader()
        for query in QUERIES:
            for item in search(query):
                row = candidate(item, query)
                if row["candidate_id"] not in existing:
                    writer.writerow(row)
                    existing.add(row["candidate_id"])

if __name__ == "__main__":
    main()

5. AI + Finance Jobs Intelligence

Jobs are not just a job board; they are demand-side evidence. If banks, funds, vendors, exchanges, insurers, and fintechs repeatedly hire for the same AI + finance roles, that tells the aggregator which workflows have budget and which skills readers should learn.

Roles to track

  • AI research analyst for buy-side or sell-side research teams.
  • Quant ML engineer and investment data scientist.
  • Financial NLP engineer for filings, news, transcripts, and compliance text.
  • Credit ML scientist for underwriting, fraud, collections, and fair lending.
  • Model risk AI specialist and compliance AI product manager.
  • Bank AI transformation lead, research copilot PM, and enterprise AI governance lead.

Job source classes

  • Company career pages for banks, funds, data vendors, exchanges, insurers, fintechs, and AI labs serving finance.
  • Public job boards such as LinkedIn, Indeed, Google Jobs, Wellfound, eFinancialCareers, Levels.fyi, and Greenhouse/Lever pages where permitted.
  • Newsletter/community job posts from quant, fintech, ML, and financial data communities.
  • Manual submissions from readers and sponsors, clearly labeled if sponsored.

# scripts/ingest_jobs.py
import csv
from datetime import datetime, timezone
from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
QUEUE = ROOT / "data" / "review_queue.tsv"
JOBS = ROOT / "data" / "jobs_ledger.tsv"

ROLE_KEYWORDS = [
    "AI finance", "financial NLP", "quant machine learning", "LLM banking",
    "model risk AI", "credit machine learning", "investment data scientist",
]

FIELDS = [
    "job_id", "company", "title", "url", "location", "role_family", "skills",
    "finance_domain", "posted_at", "discovered_at", "status", "sponsor_status", "notes"
]

def normalize_job(raw: dict[str, str]) -> dict[str, str]:
    title = raw["title"]
    company = raw["company"]
    return {
        "job_id": f"job-{company}-{title}".lower().replace(" ", "-")[:96],
        "company": company,
        "title": title,
        "url": raw["url"],
        "location": raw.get("location", "unknown"),
        "role_family": raw.get("role_family", "unknown"),
        "skills": raw.get("skills", ""),
        "finance_domain": raw.get("finance_domain", "finance"),
        "posted_at": raw.get("posted_at", ""),
        "discovered_at": datetime.now(timezone.utc).isoformat(),
        "status": "candidate",
        "sponsor_status": "organic",
        "notes": "Review for AI+finance specificity, freshness, seniority, and skill signal.",
    }

def main() -> None:
    JOBS.parent.mkdir(parents=True, exist_ok=True)
    # First implementation can read a manually exported CSV from permitted job sources.
    # Later implementations can add API/feed adapters per source terms.
    input_path = ROOT / "data" / "job_candidates_manual.csv"
    if not input_path.exists():
        print("No manual job candidate CSV found; no-op.")
        return
    existing = set()
    if JOBS.exists():
        with JOBS.open(newline="") as f:
            existing = {r["job_id"] for r in csv.DictReader(f, delimiter="	")}
    with input_path.open(newline="") as src, JOBS.open("a", newline="") as dst:
        reader = csv.DictReader(src)
        writer = csv.DictWriter(dst, fieldnames=FIELDS, delimiter="	")
        if not existing:
            writer.writeheader()
        for raw in reader:
            row = normalize_job(raw)
            if row["job_id"] not in existing:
                writer.writerow(row)
                existing.add(row["job_id"])

if __name__ == "__main__":
    main()

export async function POST(request: NextRequest) {
  const payload = await request.json();
  const parsed = AnalyticsEventSchema.safeParse(payload);

6.4 Browser event client

// components/AnalyticsTracker.tsx
'use client';

import { useEffect, useRef } from 'react';

function id(key: string): string {
  const existing = localStorage.getItem(key);
  if (existing) return existing;
  const next = crypto.randomUUID();
  localStorage.setItem(key, next);
  return next;
}

function send(event: Record<string, string | number | undefined>) {
  const anonymousId = id('finance-ai-radar-anonymous-id');
  const sessionId = id('finance-ai-radar-session-id');
  navigator.sendBeacon?.('/api/events', JSON.stringify({
    anonymousId,
    sessionId,
    path: location.pathname + location.hash,
    referrer: document.referrer,
    createdAt: new Date().toISOString(),
    ...event,
  }));
}

export function AnalyticsTracker() {
  const startedAt = useRef(Date.now());
  const maxScroll = useRef(0);

  useEffect(() => {
    send({ eventType: 'page_view' });

    const onScroll = () => {
      const height = document.documentElement.scrollHeight - window.innerHeight;
      const pct = height <= 0 ? 100 : Math.round((window.scrollY / height) * 100);
      maxScroll.current = Math.max(maxScroll.current, pct);
    };

    const onBeforeUnload = () => {
      send({
        eventType: 'session_end',
        dwellMs: Date.now() - startedAt.current,
        scrollDepth: maxScroll.current,
      });
    };

    window.addEventListener('scroll', onScroll, { passive: true });
    window.addEventListener('beforeunload', onBeforeUnload);
    return () => {
      window.removeEventListener('scroll', onScroll);
      window.removeEventListener('beforeunload', onBeforeUnload);
    };
  }, []);

  return null;
}

7. Daily Reflection Agent and Implementation Agent Loop

The loop should convert metrics into operating knowledge, then into tested code changes. Default to the smallest change that can validate a hypothesis, but allow larger implementation work when metrics show a structural product gap, such as needing a jobs vertical, dataset pages, or a major ranking/navigation redesign.

7.2 Metrics summarizer script

# scripts/summarize_metrics.py
import csv
import json
from collections import Counter, defaultdict
from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
EVENTS = ROOT / "data" / "analytics_events.tsv"
OUTPUT = ROOT / "data" / "daily_metrics.json"

MIN_DWELL_MS = 15_000

def main() -> None:
    if not EVENTS.exists():
        OUTPUT.write_text(json.dumps({"status": "no_events"}, indent=2))
        return

    clicks = Counter()
    source_clicks = Counter()
    theme_clicks = Counter()
    sessions = set()
    dwell_by_path = defaultdict(list)
    scroll_depths = []

    with EVENTS.open(newline="") as f:
        for row in csv.DictReader(f, delimiter="\t"):
            sessions.add(row.get("session_id", ""))
            event_type = row.get("event_type")
            if event_type == "source_click":
                clicks[row.get("item_id", "")] += 1
                source_clicks[row.get("source_type", "")] += 1
                theme_clicks[row.get("theme", "")] += 1
            if event_type == "session_end":
                dwell = int(row.get("dwell_ms") or 0)
                dwell_by_path[row.get("path", "/")].append(dwell)
                if row.get("scroll_depth"):
                    scroll_depths.append(int(row["scroll_depth"]))

    def average(values: list[int]) -> int:
        return round(sum(values) / len(values)) if values else 0

    summary = {
        "sessions": len([s for s in sessions if s]),
        "total_source_clicks": sum(clicks.values()),
        "top_items": clicks.most_common(10),
        "top_source_types": source_clicks.most_common(10),
        "top_themes": theme_clicks.most_common(10),
        "average_scroll_depth": average(scroll_depths),
        "high_intent_paths": [
            {"path": path, "avg_dwell_ms": average(values), "sessions": len(values)}
            for path, values in dwell_by_path.items()
            if average(values) >= MIN_DWELL_MS
        ],
    }
    OUTPUT.write_text(json.dumps(summary, indent=2))
    print(json.dumps(summary, indent=2))

if __name__ == "__main__":
    main()

10. Acceptance Criteria

Workflow and review queue

  • New discovered items land in review_queue.tsv, not directly on homepage.
  • Review page renders candidates with status, source type, URL, relevance reason, and license guess.
  • Tests prevent publishable items without license note and caveat.

Datasets

  • Kaggle and Hugging Face dataset candidates can be ingested.
  • At least 5 reviewed public finance/NLP datasets are published.
  • Dataset cards show access mode, license, NLP use case, and leakage risk.

Hugging Face intelligence

  • Hugging Face datasets, models, and papers are first-class source types.
  • Model cards show task, family, license, deployment fit, and model risks.
  • Paper cards show linked assets, method type, reproducibility, and finance domains.

Jobs intelligence

  • AI + finance jobs are tracked as a first-class source type.
  • Job posts include role family, skills, finance domain, location, freshness, and sponsor status.
  • Hiring signals connect back to datasets, papers, repos, and learning paths.

Analytics

  • Page views, source clicks, filter changes, dwell time, scroll depth, sponsor clicks, and newsletter intent are logged.
  • No raw PII or raw IP addresses are stored.
  • Daily metrics can be summarized from the event ledger.

Closed loop

  • Reflection cron updates only permitted sections of AGENTS.md.
  • Implementation cron executes at most one metric-backed change.
  • Every automated change passes lint, tests, typecheck, build, and logs a reflection row.
