4. Part 1 — Add a post-05:00 SGT queue-drain job

Design principle

The queue should not be “cleared” by deletion. It should be drained by status transitions and archival rows. Deleting candidates destroys provenance and makes it harder to learn which sources were rejected, published, stale, or duplicated.

Desired behavior

  1. Run after the 05:00 update/deploy job, e.g. 05:45 SGT.
  2. Read data/review_queue.tsv, data/sourceItems.ts, and data/source_ledger.tsv.
  3. Classify rows into:
    • published: candidate ID or URL already appears in sourceItems.ts / source_ledger.tsv.
    • rejected: manually rejected rows stay recorded but can be archived.
    • stale: candidate/triaged rows older than threshold, e.g. 30 days, that still have no publication decision.
    • active: candidate/triaged rows still worth reviewing.
  4. Write drained rows to data/review_archive.tsv with a drained_at timestamp and drain_reason.
  5. Rewrite data/review_queue.tsv with header + only active rows.
  6. Append a row to data/daily_reflections.tsv summarizing queue size before/after.
  7. Run tests/typecheck/build. If successful and source changes exist, deploy or rely on next day's deploy. Recommended: deploy immediately if the public /review page changes.
Important: current tests expect at least four queue candidates. The implementation must update tests to reflect archival behavior instead of assuming the queue is perpetually non-empty.

5. Part 2 — Move canonical hosting to keep the self-learning loop alive

Why GitHub Pages breaks the loop

The current code has app/api/events/route.ts, but GitHub Pages serves static files only. It cannot execute a Next route handler. Therefore browser events from AnalyticsTracker cannot be written to data/analytics_events.tsv on the live site. Without a live event collector, daily_metrics.json remains seeded/manual and the self-learning loop is not behavior-grounded.

8.2 lib/library.ts freshness helpers

import type { ContentItem, DateWindow, SearchFilters } from './types';

const MS_PER_DAY = 24 * 60 * 60 * 1000;

export function daysForWindow(dateWindow: DateWindow, customDays?: number): number | null {
  if (dateWindow === 'all') return null;
  if (dateWindow === '7d' || dateWindow === 'week') return 7;
  if (dateWindow === 'month') return 30;
  if (dateWindow === 'year') return 365;
  if (dateWindow === 'custom') {
    const value = Math.floor(customDays ?? 30);
    return Math.min(3650, Math.max(1, value));
  }
  return null;
}

export function isWithinDateWindow(publishedAt: string, dateWindow: DateWindow, customDays: number | undefined, now = new Date()): boolean {
  const days = daysForWindow(dateWindow, customDays);
  if (days === null) return true;
  const published = new Date(`${publishedAt}T00:00:00.000Z`);
  if (Number.isNaN(published.getTime())) return false;
  const ageDays = Math.floor((now.getTime() - published.getTime()) / MS_PER_DAY);
  return ageDays >= 0 && ageDays <= days;
}

export function searchItems(items: ContentItem[], filters: SearchFilters, now = new Date()): ContentItem[] {
  const query = filters.query.trim().toLowerCase();

  return items
    .filter((item) => filters.sourceType === 'all' || item.sourceType === filters.sourceType)
    .filter((item) => filters.theme === 'all' || item.themes.includes(filters.theme))
    .filter((item) => isWithinDateWindow(item.publishedAt, filters.dateWindow, filters.customDays, now))
    .filter((item) => {
      if (!query) return true;
      // existing searchable-field logic stays here
      return searchable.includes(query);
    })
    .sort((a, b) => b.score - a.score || a.title.localeCompare(b.title));
}

8.5 Queue-drain script

# scripts/drain_review_queue.py
import argparse
import csv
from datetime import datetime, timezone
from pathlib import Path

ROOT = Path(__file__).resolve().parents[1]
QUEUE = ROOT / 'data' / 'review_queue.tsv'
ARCHIVE = ROOT / 'data' / 'review_archive.tsv'
SOURCE_LEDGER = ROOT / 'data' / 'source_ledger.tsv'
SOURCE_ITEMS = ROOT / 'data' / 'sourceItems.ts'
FIELDS = ['candidate_id', 'source_type', 'title', 'url', 'publisher', 'discovered_at', 'relevance_reason', 'license_guess', 'status', 'reviewer_notes']
ARCHIVE_FIELDS = FIELDS + ['drained_at', 'drain_reason']


def read_tsv(path: Path) -> list[dict[str, str]]:
    if not path.exists():
        return []
    with path.open(newline='') as f:
        return list(csv.DictReader(f, delimiter='\t'))


def write_tsv(path: Path, rows: list[dict[str, str]], fields: list[str]) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    with path.open('w', newline='') as f:
        writer = csv.DictWriter(f, fieldnames=fields, delimiter='\t')
        writer.writeheader()
        writer.writerows(rows)


def published_markers() -> set[str]:
    markers: set[str] = set()
    if SOURCE_ITEMS.exists():
        text = SOURCE_ITEMS.read_text()
        markers.update(line.strip() for line in text.splitlines() if 'id:' in line or 'url:' in line)
    for row in read_tsv(SOURCE_LEDGER):
        markers.add(row.get('item_id', ''))
        markers.add(row.get('url', ''))
    return {marker for marker in markers if marker}


def row_age_days(row: dict[str, str], now: datetime) -> int:
    raw = row.get('discovered_at', '')
    try:
        discovered = datetime.fromisoformat(raw.replace('Z', '+00:00'))
    except ValueError:
        return 9999
    return (now - discovered).days


def drain_reason(row: dict[str, str], markers: set[str], stale_days: int, now: datetime) -> str | None:
    if row.get('status') == 'published':
        return 'status_published'
    if row.get('status') == 'rejected':
        return 'status_rejected'
    if row.get('candidate_id') in markers or row.get('url') in markers:
        return 'already_in_source_library'
    if row.get('status') in {'candidate', 'triaged'} and row_age_days(row, now) > stale_days:
        return f'stale_gt_{stale_days}_days'
    return None


def main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument('--stale-days', type=int, default=30)
    args = parser.parse_args()

    now = datetime.now(timezone.utc)
    rows = read_tsv(QUEUE)
    markers = published_markers()
    active: list[dict[str, str]] = []
    archived: list[dict[str, str]] = []

    for row in rows:
        reason = drain_reason(row, markers, args.stale_days, now)
        if reason:
            archived.append({**row, 'drained_at': now.isoformat(), 'drain_reason': reason})
        else:
            active.append(row)

    existing_archive = read_tsv(ARCHIVE)
    write_tsv(ARCHIVE, existing_archive + archived, ARCHIVE_FIELDS)
    write_tsv(QUEUE, active, FIELDS)
    print(f'queue_before={len(rows)} queue_after={len(active)} archived={len(archived)}')


if __name__ == '__main__':
    main()

8.6 DB-backed analytics insert for Vercel/Neon

// lib/analyticsDb.ts
import { neon } from '@neondatabase/serverless';
import type { AnalyticsEvent } from './analytics';

const databaseUrl = process.env.DATABASE_URL;

export async function insertAnalyticsEvent(event: AnalyticsEvent): Promise<void> {
  if (!databaseUrl) {
    throw new Error('DATABASE_URL is not configured');
  }
  const sql = neon(databaseUrl);
  await sql`
    insert into analytics_events (
      event_type, anonymous_id, session_id, path, item_id, source_type, theme, query,
      date_window, custom_days, rank_position, job_id, role_family, finance_domain,
      location, dwell_ms, scroll_depth, referrer, created_at
    ) values (
      ${event.eventType}, ${event.anonymousId}, ${event.sessionId}, ${event.path},
      ${event.itemId ?? null}, ${event.sourceType ?? null}, ${event.theme ?? null}, ${event.query ?? null},
      ${event.dateWindow ?? null}, ${event.customDays ?? null}, ${event.rankPosition ?? null},
      ${event.jobId ?? null}, ${event.roleFamily ?? null}, ${event.financeDomain ?? null},
      ${event.location ?? null}, ${event.dwellMs ?? null}, ${event.scrollDepth ?? null},
      ${event.referrer ?? null}, ${event.createdAt}
    )`;
}

8.7 Runtime API route

// app/api/events/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { AnalyticsEventSchema } from '../../../lib/analytics';
import { insertAnalyticsEvent } from '../../../lib/analyticsDb';

export async function POST(request: NextRequest) {
  let payload: object;
  try {
    payload = JSON.parse(await request.text());
  } catch {
    return NextResponse.json({ ok: false, error: 'invalid_json' }, { status: 400 });
  }

  const parsed = AnalyticsEventSchema.safeParse(payload);
  if (!parsed.success) {
    return NextResponse.json({ ok: false, error: 'invalid_event' }, { status: 400 });
  }

  await insertAnalyticsEvent(parsed.data);
  return NextResponse.json({ ok: true });
}

11. Acceptance criteria

  • Queue drain: a 05:45 SGT Hermes cron job exists, is enabled, uses project workdir, and archives queue rows without deleting provenance.
  • Queue archive: data/review_archive.tsv exists after first drain and includes drained_at and drain_reason.
  • No silent clearing: active candidates remain in review_queue.tsv; published/rejected/stale rows are auditable in archive.
  • Runtime metrics: live production site can POST page_view, source_click, filter_change, and session_end to a functioning endpoint.
  • Durable analytics: events persist in a database or equivalent durable store, not serverless local files.
  • Self-learning continuity: 03:00 job reads a real metrics export/summary and writes/uses daily_metrics.json.
  • Freshness filter: users can filter by past 7 days, past week, past month, past year, all time, and custom past X days.
  • Verification: npm test, npm run typecheck, and build pass.
  • Live smoke test: canonical production URL returns 200 and browser console is clean enough for normal operation.

12. Rollback plan

ChangeRollback
Queue-drain cron causes bad archivalPause/remove the 05:45 cron job. Restore review_queue.tsv rows from review_archive.tsv by filtering affected drained_at. Because no rows are deleted, recovery is straightforward.
Runtime hosting deploy failsKeep GitHub Pages as current production until Vercel/Cloudflare passes smoke tests. Do not change canonical links until runtime analytics is verified.
Analytics DB failsDisable NEXT_PUBLIC_ANALYTICS_ENDPOINT or make API return 202/no-op temporarily. The site should still render without analytics.
Date filter hides too many itemsChange default from month back to all; keep the filter available for users who want recency.
Static mirror breaksKeep runtime host canonical; fix static export separately or retire GitHub Pages mirror.

13. Final notes for implementer

  • Do not treat queue draining as deletion. Archive first, rewrite active queue second.
  • Do not claim the self-learning loop works until a live event reaches durable storage and appears in a daily metrics summary.
  • Do not overfit ranking to seeded metrics. Wait for real click/dwell/filter/source/job/sponsor events.
  • Keep daily_reflections.tsv as the human-auditable place for hypotheses, decisions, files changed, verification, and rollback notes.
  • If runtime hosting is Vercel, update documentation to say GitHub Pages is no longer canonical unless deliberately kept as a static mirror.
